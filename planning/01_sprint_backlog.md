# Sprint Backlog

## Overview

The development of PharmaVision BI follows an iterative approach inspired by Scrum.

Each sprint delivers a functional increment of the product, allowing continuous validation, documentation, and improvement.

The objective is not only to build a dashboard but to document the complete development process of a Business Intelligence solution.

# Sprint Principles

Every sprint should produce a functional increment.

Priority order:

1. Working software
2. Clean architecture
3. Documentation
4. Visual polish

The project should remain deployable and demonstrable after every completed sprint.

---

# Definition of Done (Project)

PharmaVision BI will be considered complete when:

- The application runs successfully.
- The dashboard is fully interactive.
- Documentation is complete.
- Code follows a modular architecture.
- Repository is portfolio-ready.
- The project can be presented confidently during technical interviews.
