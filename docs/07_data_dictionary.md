# Data Dictionary

## Overview

This document defines the structure of the synthetic dataset used in PharmaVision BI.

The dataset simulates commercial sales transactions for a pharmaceutical company and has been designed to support Business Intelligence analysis, KPI calculation, and interactive dashboard development.

All data is synthetic and does not represent any real company, customer, or commercial operation.

---

# Dataset Information

| Property | Value |
|----------|-------|
| Dataset Name | sales_data.parquet |
| Granularity | One row per sales transaction |
| Storage Format | Apache Parquet |
| Estimated Records | 100,000+ |
| Update Frequency | Static (Portfolio Project) |

---

# Entity Relationship

The dataset follows the following business hierarchy:

Region

↓

District

↓

Route

↓

Customer

↓

Product

Each sales transaction belongs to a single customer, product, route, district, and region.

---

# Business Definitions

## Revenue

Total monetary value generated by a sales transaction.

Formula:

Revenue = Units × Unit Price

---

## Units

Number of product units sold during the transaction.

---

## Region

Highest level of the commercial hierarchy.

---

## District

Commercial subdivision within a region.

---

## Route

Operational sales territory managed by a sales representative.

---

## Category

Business grouping of products.

Examples:

- Respiratory
- Cardiology
- Gastroenterology
- Pain Management

---

# Data Quality Rules

The dataset should satisfy the following validations:

| Rule | Description |
|------|-------------|
| No duplicate transaction_id | Every transaction must be unique |
| No null dates | Every sale requires a valid date |
| Revenue ≥ 0 | Revenue cannot be negative |
| Units > 0 | At least one unit must be sold |
| Valid hierarchy | Every district belongs to exactly one region |
| Product category required | Every product belongs to one category |
